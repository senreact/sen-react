# Le parcours, phase par phase

Sept phases. Chaque phase produit un résultat visible et fonctionnel. Vous voyez la progression en continu — rien n'est caché jusqu'à une grande révélation finale.

## Phase 0 — Fondations

Nous mettons en place la tuyauterie : le dépôt de code, la base de données, le système de gestion de contenu, l'environnement d'hébergement. Aucune fonctionnalité n'est encore construite. À la fin de cette phase, il existe un squelette de site bilingue vide, vivant sur internet, avec un tableau de bord d'administration dans lequel votre équipe peut se connecter.

**Ce que vous voyez** : une URL en ligne, un site vide, une connexion admin.

## Phase 1 — Tableau de bord des opportunités

La première vraie fonctionnalité. Votre équipe REACT peut ajouter des opportunités via l'admin ; les visiteurs peuvent les parcourir, les rechercher, les filtrer. Le pipeline d'agrégation automatique est construit mais démarre avec une **petite liste de sources validées** que vous nous confirmerez.

**Ce que vous voyez** : des opportunités affichées, l'équipe admin qui les saisit, une file d'attente de validation pour les entrées automatiques.

## Phase 2 — Annuaire B2B

Fiches d'entreprises avec filtres par secteur et région. Formulaire d'auto-soumission pour les entrepreneurs. File d'attente de validation.

**Ce que vous voyez** : l'annuaire qui grandit à mesure que les entreprises s'inscrivent ; les coordonnées apparaissant publiquement.

## Phase 3 — Pôle de contenu

Articles, programmes REACT (vos six projets), supports de formation, histoires de réussite. Votre équipe peut écrire, modifier, publier et traduire le contenu. Les visiteurs lisent dans la langue de leur choix.

**Ce que vous voyez** : la voix de REACT en ligne, entièrement bilingue, entièrement sous votre contrôle.

## Phase 4 — Expérience membre

Les membres s'inscrivent avec e-mail ou téléphone + mot de passe. Ils peuvent enregistrer des opportunités, suivre des programmes, recevoir des notifications sur les nouvelles correspondances.

**Ce que vous voyez** : les premiers membres REACT qui s'inscrivent.

## Phase 5 — Engagement citoyen

La section de la plateforme où les citoyens contribuent — nous définissons cela ensemble pendant la découverte. Idées, suggestions, commentaires, participation à des initiatives. Forme exacte à préciser avec vous.

**Ce que vous voyez** : les citoyens qui interagissent publiquement avec le travail de REACT.

## Phase 6 — Lancement restreint

Lancement sur invitation à un petit groupe — équipe REACT, partenaires, quelques entrepreneurs de confiance. Ils utilisent la plateforme, nous corrigeons les aspérités, nous affinons l'expérience.

**Ce que vous voyez** : la plateforme utilisée par de vraies personnes pour la première fois ; les retours collectés et traités.

## Phase 7 — Lancement public

Campagne de communication, annonce sur les réseaux sociaux, communiqué de presse. On passe au public.

**Ce que vous voyez** : le trafic, les inscriptions, les opportunités qui trouvent preneur, les entreprises qui sont contactées.

## Après le lancement — Propriété continue

Nous restons disponibles pour les questions. Votre équipe fait tourner la plateforme. Nous fournissons :

- Un guide utilisateur en français pour les tâches courantes
- Des sessions de formation pour l'équipe REACT
- Un appui aux questions qui apparaissent
- Un parcours clair vers l'auto-maintenance assistée par Claude dès que vous serez prêts

Nous ne disparaissons pas — nous transmettons.

## Ce que nous ne ferons PAS (volontairement)

- **Pas de MVP bâclé** — chaque fonctionnalité est construite proprement, pas mise de côté pour plus tard
- **Pas de dépendance bloquante à un fournisseur** — vous pouvez reprendre le code et l'héberger ailleurs
- **Pas de coûts cachés** — vous voyez chaque coût avant que nous nous engagions
- **Pas de dette technique déversée sur REACT** — nous documentons tout ce que nous construisons

## Ce que nous demandons à REACT pendant le chantier

- La disponibilité d'Amadou (ou d'une personne référente désignée) pendant ~30 minutes toutes les 1–2 semaines pour examiner l'avancement
- Des réponses aux questions de clarification dans un délai raisonnable (WhatsApp convient)
- Des retours honnêtes quand quelque chose paraît incorrect ou ne vous convient pas
- L'autorisation de vous montrer des versions brutes très tôt — c'est ainsi que nous évitons de construire la mauvaise chose

## Note sur le calendrier

Nous ne vous donnons volontairement pas de date fixe « ce sera terminé pour le X » dans ce document. Les estimations faites en amont sont presque toujours fausses et mettent la qualité sous pression. À la place, nous nous engageons sur :

1. **La transparence** — vous saurez toujours sur quoi nous travaillons et ce qui vient ensuite
2. **Une progression visible régulière** — quelque chose de nouveau à regarder toutes les 1–2 semaines
3. **Des mises à jour honnêtes** — quand quelque chose prend plus de temps, vous en entendez la raison, pas des excuses
