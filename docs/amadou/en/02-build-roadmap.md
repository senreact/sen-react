# The journey, phase by phase

Seven phases. Each phase produces a working, visible outcome. You see progress along the way — nothing is hidden until a big reveal at the end.

## Phase 0 — Foundations

We set up the plumbing: the repository, the database, the content management system, the hosting environment. We do not build any features yet. At the end of this phase, there is a blank bilingual website skeleton that lives on the internet, with an admin dashboard that your team can log into.

**You see**: a live URL, an empty site, an admin login.

## Phase 1 — Opportunities Dashboard

The first real feature. Your REACT team can add opportunities through the admin; visitors can browse, search, and filter them. The automatic aggregation pipeline is built but starts with a **small list of approved sources** that you confirm with us.

**You see**: opportunities being displayed, admin team adding them, an approval queue for automatic entries.

## Phase 2 — B2B Directory

Business listings with sector and region filters. Self-submission form for business owners. Admin approval queue.

**You see**: the directory growing as businesses submit themselves; contact details appearing publicly.

## Phase 3 — Content Hub

Articles, REACT programmes (your six projects), training material, success stories. Your team can write, edit, publish, and translate content. Visitors can read in their preferred language.

**You see**: REACT's voice online, fully bilingual, fully under your control.

## Phase 4 — Member Experience

Members sign up with email or phone + password. They can save opportunities, follow programmes, receive notifications on new matches.

**You see**: the first REACT members registering.

## Phase 5 — Citizen Engagement

The section of the platform where citizens contribute — we shape this together during discovery. Ideas, suggestions, comments, participation in initiatives. Exact shape to be defined with you.

**You see**: citizens interacting with REACT's work publicly.

## Phase 6 — Soft launch

Invite-only launch to a small group — REACT team, partners, a few trusted entrepreneurs. They use the platform, we fix anything rough, we refine the experience.

**You see**: the platform used by real humans for the first time; feedback collected and addressed.

## Phase 7 — Public launch

Communication campaign, social media announcement, press release. Go public.

**You see**: traffic, sign-ups, opportunities being filled, businesses being contacted.

## After launch — Ongoing ownership

We remain available for questions. Your team runs the platform. We provide:

- Support for questions that arise
- A clear path to Claude-assisted self-maintenance when you are ready

We are not disappearing — we are handing over.

## What we will NOT do (on purpose)

- **No rushed MVP** — every feature gets built properly, not parked for later
- **No vendor lock-in** — you can take the code and host it anywhere
- **No hidden costs** — you see every cost before we commit to it
- **No technical debt dumped on REACT** — we document everything we build

## What we ask from REACT during the build

- Answers to clarifying questions
- Honest feedback when something looks wrong or feels wrong
- Permission to show you rough versions early — this is how we avoid building the wrong thing

## Timing note

We deliberately do not give you a fixed "this will be done by X date" timeline in this document. Software estimates made up front are almost always wrong, and they put pressure on quality. Instead, we commit to:

1. **Transparency** — you will always know what we are working on and what's coming next
2. **Regular visible progress** — something new to look at every 1–2 weeks
3. **Honest updates** — when something takes longer, you hear why, not excuses
